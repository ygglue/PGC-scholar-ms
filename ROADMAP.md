# Roadmap

Implementation priority based on gaps between PGC_SPEC.md and current system.

## Phase 1 — Evaluation Remarks

The PGC_SPEC dedicates a full section to evaluation remarks. **Done.**

### Backend
- `backend/app/models/evaluation_remark.py` — Model with FK to pending_changes, users, scholars
- `backend/app/routers/remarks.py` — `GET /remarks/me` (scholar), `GET /remarks/pending-change/{id}` (evaluator), `POST /remarks/` (evaluator create)
- `backend/migrations/versions/fc4b66102c67_add_evaluation_remarks.py` — Migration (autogenerated via alembic)

### Evaluator-Tauri
- `PendingSubmissions.tsx` — Remarks History panel in the review sidebar. Shows remark history per submission. The "Evaluator Notes" textarea doubles as the remark input — text is saved as a remark automatically when the review is submitted.

### Scholar-Web
- `Inbox.jsx` — Replaced hardcoded "No Remarks Yet" stub. Fetches from `/remarks/me`. Each remark card shows: evaluator name, remark type badge, linked submission type + status, submission date, remark text.

## Phase 2 — Scholar Read-Only Views

The backend models exist, but scholars have no way to view their program history, CSR events, or allowance records in the web app.

- [ ] Program shift history display on scholar-web Profile page
- [ ] CSR events attended display on scholar-web (new page or section)
- [ ] Allowance records display on scholar-web
