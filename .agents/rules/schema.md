# Scholar Management System — Database Schema

**Database:** PostgreSQL (hosted on Supabase)  
**ORM:** SQLAlchemy  
**Migration tool:** Alembic

---

## Tables overview

| Table | Description |
|---|---|
| `users` | Login accounts for all roles |
| `scholars` | Full scholar profile linked to a user |
| `academic_records` | One row per scholar per semester |
| `prospectus_grades` | One row per subject per semester |
| `documents` | Uploaded files (COR, ROG, letters, etc.) |
| `program_history` | Tracks every course/school shift |
| `pending_changes` | Approval queue for all scholar-submitted edits |
| `announcements` | Broadcast messages from evaluators to scholars |
| `announcement_receipts` | Tracks delivery and read status per scholar |

---

## `users`

## `documents`

Uploaded files stored in Supabase Storage (private bucket). Access controlled via signed URLs generated by the backend.

| Column | Type | Constraints | Notes |
|---|---|---|---|
| `id` | UUID | PK, default uuid4 | |
| `scholar_id` | UUID | FK → scholars.id, NOT NULL | |
| `academic_record_id` | UUID | FK → academic_records.id, nullable | optional link to a semester |
| `doc_type` | VARCHAR | NOT NULL | `COR`, `ROG`, `explanation_letter`, `completion_form`, `other` |
| `file_name` | VARCHAR | NOT NULL | original filename shown to user |
| `storage_path` | TEXT | NOT NULL | path inside Supabase Storage bucket |
| `file_size` | INTEGER | nullable | bytes |
| `mime_type` | VARCHAR | nullable | e.g. `image/jpeg`, `application/pdf` |
| `is_verified` | BOOLEAN | default false | evaluator confirmed the document |
| `uploaded_at` | TIMESTAMP | server default now() | |

**Storage path format:** `scholars/{scholar_id}/{doc_type}/{uuid}_{filename}`  
**Allowed file types:** JPEG, PNG, PDF  
**Max file size:** 10MB  
**Signed URL expiry:** 120 seconds

## `pending_changes`

The approval queue. Every edit a scholar submits — profile, grades, or documents — creates a row here. Evaluators review, approve, reject, or request more info.

| Column | Type | Constraints | Notes |
|---|---|---|---|
| `id` | UUID | PK, default uuid4 | |
| `scholar_id` | UUID | FK → scholars.id, NOT NULL | |
| `submitted_by` | UUID | FK → users.id, NOT NULL | the scholar's user id |
| `reviewed_by` | UUID | FK → users.id, nullable | evaluator who reviewed |
| `claimed_by` | UUID | FK → users.id, nullable | evaluator currently reviewing (lock) |
| `claimed_at` | TIMESTAMP | nullable | lock expires after 30 minutes |
| `assigned_to` | UUID | FK → users.id, nullable | optional assignment to specific evaluator |
| `change_type` | VARCHAR | NOT NULL | `profile`, `grades`, `documents` |
| `payload` | JSONB | NOT NULL | the actual changes as a JSON diff |
| `status` | VARCHAR | default "pending" | `pending`, `approved`, `rejected`, `more_info` |
| `submitted_at` | TIMESTAMP | server default now() | |
| `reviewed_at` | TIMESTAMP | nullable | |
| `evaluator_note` | TEXT | nullable | remark attached by evaluator |

**Payload format for profile changes:**
```json
{
  "first_name": { "from": "Juan", "to": "John" },
  "address": { "from": "Old St.", "to": "New Ave." }
}
```

**Payload format for grade changes:**
```json
{
  "academic_record_id": "uuid-here",
  "school_year": "2024-2025",
  "semester": "1st",
  "remarks_status": "complete",
  "grade_count": 8
}
```

**Concurrency locking:** When an evaluator opens a submission, `claimed_by` is set to their user ID and `claimed_at` is set to now. Any other evaluator trying to open the same submission within 30 minutes receives a 409 conflict. The lock auto-expires after 30 minutes.

## `announcements`

Broadcast messages from evaluators to scholars. Can be filtered by batch, school, or status.

| Column | Type | Constraints | Notes |
|---|---|---|---|
| `id` | UUID | PK, default uuid4 | |
| `created_by` | UUID | FK → users.id, NOT NULL | evaluator who created it |
| `title` | VARCHAR | NOT NULL | |
| `message` | TEXT | NOT NULL | |
| `type` | VARCHAR | default "general" | `general`, `reminder`, `warning`, `deadline` |
| `recipient_filter` | JSONB | nullable | null = send to all scholars |
| `created_at` | TIMESTAMP | server default now() | |
| `scheduled_at` | TIMESTAMP | nullable | for future scheduling; null = send immediately |

**Recipient filter format:**
```json
{ "batch": "Batch 5", "school": "UST", "status": "active" }
```

## Notes

- All primary keys are UUID v4, generated by the application layer.
- All timestamps use `server_default=func.now()` so the database sets them automatically.
- `hashed_password` is nullable because scholars log in via Google OAuth — the password is managed by Google, not this system.
- Documents are never stored in the database. Only the `storage_path` (the path inside Supabase Storage) is stored. The backend generates a signed URL on demand that expires in 120 seconds.
- The `pending_changes.payload` column uses JSONB so it can store any shape of change diff regardless of which table is being modified.
- `program_history.end_date = null` means that row represents the scholar's current active program.
